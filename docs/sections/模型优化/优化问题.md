## 一、优化问题

在深度学习中，梯度下降是一种常用的优化方法，用于最小化损失函数并优化模型参数。然而，梯度下降也可能会遇到一些挑战，例如局部最小值问题，导致无法达到理想的损失水平。如果用图形表示，模型可以表示的函数集合形成的蓝色区域确实包含了一些损失较低的函数。然而，梯度下降算法可能无法找到这些损失较低的函数，它只是解决了优化问题并找到了某个参数 $ \theta^* $ 就停止了。

这就像是在大海中捞针，针确实在海里，但我们却无法将其捞起。当训练数据的损失不够低时，我们需要区分是模型偏差（model bias）还是优化问题。如果找不到损失低的函数，可能是因为模型的灵活性不够，即“海里没有针”，或者是模型的灵活性已经足够，但梯度下降优化不力，无法将针捞起来。

### 1、通过比较获得洞察

为了判断模型是否足够大，一个建议的方法是比较不同模型的表现。例如，我们可以比较20层和56层的残差网络（residual network）在测试集上的表现。根据2015年的一篇论文（链接：http://arxiv.org/abs/1512.03385），尽管56层网络在测试集上的损失比20层网络高，但这并不是过拟合（overfitting）。实际上，这表明56层网络的优化没有做好，因为如果优化成功，56层网络应该能够达到比20层网络更低的损失。

### 2、从浅层网络开始

面对一个新问题时，建议先尝试一些较小或较浅的网络，或者使用深度学习之外的方法，如线性模型或支持向量机。这些模型通常更容易优化，不太可能出现优化失败的问题。通过训练这些简单的模型，你可以先了解它们能达到的损失水平。

### 3、如果深层网络在训练数据上没有获得更小的损失，则存在优化问题

如果你发现深层模型与浅层模型相比，尽管具有更大的灵活性，但损失并没有更低，这表明优化存在问题，梯度下降可能不足够有效。在这种情况下，你需要寻找其他方法来改善优化过程。

例如，在观看人数预测的例子中，随着网络层数的增加，损失本应逐渐降低。但如果增加到5层时损失反而增大，这显然不是模型偏差问题，而是优化问题。

如果优化做得不好，我们将在下一节课中讨论如何解决这个问题。现在，你需要知道如何判断训练损失较大时是模型偏差还是优化问题。如果是模型偏差，可以通过增大模型来解决；如果是优化失败，那么需要参考后续课程来解决这个问题。

### 4、测试数据上的损失

当你通过努力使训练数据的损失变小后，接下来可以观察测试数据上的损失。如果测试数据的损失也很小，并且比强基线（strong baseline）还要小，那么就没有必要再做进一步的工作。但如果训练数据上的损失小而测试数据上的损失大，那么你可能真的遇到了过拟合问题。请注意，只有当训练损失小而测试损失大时，才称之为过拟合。许多同学在测试结果不佳时就认为是过拟合，但并非总是如此。

当你想要改进结果时，首先要检查训练数据上的损失。确保优化没有问题，模型足够大，然后才考虑是否是测试数据的问题。如果是训练损失小而测试损失大，那么可能是过拟合问题。

## 二、超参数

超参数是机器学习算法中的参数，但不同于模型参数（如权重和偏置），它们不是通过训练数据来学习的，而是在训练之前手动设置的。超参数的选择通常是基于经验、启发式方法或者试验来进行的。

### 1、常见的超参数

#### （1）学习率（Learning Rate）

学习率是控制模型在每次迭代中更新参数的步长大小。过小的学习率会导致收敛速度过慢，而过大的学习率可能会导致无法收敛。通常需要通过实验来调整学习率的大小。

#### （2）正则化参数（Regularization Parameter）

正则化参数用于控制模型的复杂度，防止过拟合。常见的正则化方法包括L1正则化和L2正则化，正则化参数的选择影响着模型的泛化能力。

#### （3）树的深度（Tree Depth）

对于决策树模型和基于树的集成模型（如随机森林、梯度提升树等），树的深度是一个重要的超参数，它控制着树的复杂度和拟合能力。

#### （4）窗口大小（Window Size）

在一些序列模型中（如循环神经网络、卷积神经网络等），窗口大小是一个重要的超参数，它影响着模型对序列数据的理解能力。

#### （5）批量大小（Batch Size）

批量大小是指在训练过程中每个小批量样本的数量，它影响着模型在训练过程中的稳定性和收敛速度。

### 2、超参数调优方法

#### （1）网格搜索（Grid Search）

网格搜索是一种常用的超参数调优方法，它通过在指定的超参数空间中进行穷举搜索来找到最优的超参数组合。

代码示例：

```python
from sklearn.model_selection import GridSearchCV

# 定义模型和超参数空间
rf = RandomForestClassifier()
param_grid = {
    'n_estimators': [50, 100, 200],
    'max_features': ['auto', 'sqrt', 'log2']
}

# 创建 GridSearchCV 对象
grid_search = GridSearchCV(estimator=rf, param_grid=param_grid, cv=5)
```

`GridSearchCV`的主要参数包括：

- `estimator`：要使用的模型，例如`RandomForestClassifier()`。
- `param_grid`：要搜索的参数网格，是一个字典或字典列表，键是参数名称，值是要尝试的参数值列表。
- `cv`：交叉验证的折数，用于评估每种参数组合的性能，默认为5折交叉验证。

在这个示例中，我们使用`RandomForestClassifier`作为模型，并定义了要搜索的`n_estimators`和`max_features`的参数网格。然后，我们创建了一个`GridSearchCV`对象，用于搜索最优的超参数组合。

#### （2）随机搜索（Random Search）

随机搜索是另一种超参数调优方法，与网格搜索不同的是，它是在指定的超参数空间中随机抽样来进行搜索，从而可以更快地找到较好的超参数组合。

#### （3）贝叶斯优化（Bayesian Optimization）

贝叶斯优化是一种基于贝叶斯统计模型的超参数调优方法，它通过不断地在超参数空间中探索，利用历史观察结果来指导下一次的超参数选择，从而高效地找到最优的超参数组合。
