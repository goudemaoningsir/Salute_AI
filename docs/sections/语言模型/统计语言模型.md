## 一、条件概率和乘法公式

在自然语言处理（NLP）中，条件概率和乘法公式（链式法则）是构建统计语言模型的基础。

### 1. 条件概率

条件概率是概率论中的一个基本概念，它描述了在某个事件已经发生的情况下，另一个事件发生的概率。具体来说，如果有两个事件A和B，且事件B发生的概率$P(B) > 0$，那么事件A在事件B发生的条件下发生的概率，即条件概率，可以表示为：
$$
P(A|B) = \frac{P(A \cap B)}{P(B)}
$$
这里的$P(A \cap B)$是事件A和事件B同时发生的概率。

### 2. 乘法公式（链式法则）

乘法公式是条件概率的直接结果，它允许我们计算两个或多个事件联合发生的概率。对于两个事件A和B，如果$P(B) > 0$，那么事件A和B同时发生的概率可以表示为：
$$
P(A \cap B) = P(A) \times P(B|A)
$$
如果有更多的事件，比如N个事件$A_1, A_2, ..., A_N$，乘法公式可以扩展为：
$$
P(A_1, A_2, ..., A_N) = P(A_1) \times P(A_2|A_1) \times ... \times P(A_N|A_1, A_2, ..., A_{N-1})
$$
这个公式被称为链式法则，因为它将联合概率分解为一系列条件概率的乘积。

## 二、序列模型

### 1. 概述

在日常生活中，许多场景都涉及序列数据。以电影评分为例，人们对电影的看法会随着时间而变化，因此可以通过时间序列模型进行更准确的预测。

### 2. 统计工具

处理序列数据需要统计工具和新的深度神经网络架构。为了简单起见，我们以股票价格（富时100指数）为例。用$x_t$表示价格，即在时间步$t$时观察到的价格$x_t$。$t$对于本文中的序列通常是离散的，并在整数或其子集上变化。

![../_images/ftse100.png](../../images/8-0658698.png)

假设一个交易员想在$t$日的股市中表现良好，于是通过以下途径预测$x_t$：

$$
x_t \sim P(x_t \mid x_{t-1}, \ldots, x_1)
$$

### 3. 自回归模型

为了实现这个预测，交易员可以使用回归模型。主要问题在于输入数据的数量，输入$x_{t-1}, \ldots, x_1$本身因$t$而异。也就是说，输入数据的数量将会随着我们遇到的数据量的增加而增加，因此需要一个近似方法来使这个计算变得容易处理。本章后面的大部分内容将围绕着如何有效估计$P(x_t \mid x_{t-1}, \ldots, x_1)$展开。

简单地说，它归结为以下两种策略：

1. **截断序列长度**：假设在现实情况下相当长的序列$x_{t-1}, \ldots, x_1$可能是不必要的，只需要满足某个长度为$\tau$的时间跨度，即使用观测序列$x_{t-1}, \ldots, x_{t-\tau}$。当下获得的最直接的好处是参数的数量总是不变的。这种模型被称为自回归模型，因为它们是对自己执行回归。

2. **使用隐藏状态**：保留一些对过去观测的总结$h_t$，并且同时更新预测$\hat{x}_t$和总结$h_t$。这就产生了基于$\hat{x}_t = P(x_t \mid h_{t})$估计$x_t$，以及公式$h_t = g(h_{t-1}, x_{t-1})$更新的模型。这类模型也被称为隐变量自回归模型。

这两种情况都有一个显而易见的问题：如何生成训练数据？一个经典方法是使用历史观测来预测下一个未来观测。尽管时间不会停滞不前，一个常见的假设是序列本身的动力学不会改变。统计学家称不变的动力学为静止。

### 4. 马尔可夫模型

在自回归模型的近似法中，我们使用$x_{t-1}, \ldots, x_{t-\tau}$而不是$x_{t-1}, \ldots, x_1$来估计$x_t$。只要这种近似是精确的，我们就说序列满足马尔可夫条件。特别是，如果$\tau = 1$，得到一个一阶马尔可夫模型，$P(x)$由下式给出：

$$
P(x_1, \ldots, x_T) = \prod_{t=1}^T P(x_t \mid x_{t-1})  \\ \text{ 当 } P(x_1 \mid x_0) = P(x_1)
$$
当假设$x_t$仅是离散值时，这样的模型特别有效，因为在这种情况下，使用动态规划可以沿着马尔可夫链精确地计算结果。例如，我们可以高效地计算$P(x_{t+1} \mid x_{t-1})$：

$$
\begin{aligned}
P(x_{t+1} \mid x_{t-1}) &= \frac{\sum_{x_t} P(x_{t+1}, x_t, x_{t-1})}{P(x_{t-1})}\\
&= \frac{\sum_{x_t} P(x_{t+1} \mid x_t, x_{t-1}) P(x_t, x_{t-1})}{P(x_{t-1})}\\
&= \sum_{x_t} P(x_{t+1} \mid x_t) P(x_t \mid x_{t-1})
\end{aligned}
$$

利用这一事实，我们只需要考虑过去观察中的一个非常短的历史：$P(x_{t+1} \mid x_t, x_{t-1}) = P(x_{t+1} \mid x_t)$。隐马尔可夫模型中的动态规划超出了本节的范围，而这些计算工具已经在控制算法和强化学习算法中广泛使用。

### 5. 因果关系

原则上，将$P(x_1, \ldots, x_T)$倒序展开也没问题。基于条件概率公式，我们总是可以写出：

$$
P(x_1, \ldots, x_T) = \prod_{t=T}^1 P(x_t \mid x_{t+1}, \ldots, x_T)
$$
然而，在许多情况下，数据存在一个自然的方向，即在时间上是前进的。未来的事件不能影响过去。因此，解释$P(x_{t+1} \mid x_t)$应该比解释$P(x_t \mid x_{t+1})$更容易。例如，对于某些可加性噪声$\epsilon$，我们可以找到$x_{t+1} = f(x_t) + \epsilon$，而反之则不行。而这个向前推进的方向恰好也是我们通常感兴趣的方向。

## 三、统计语言模型

统计语言模型（Statistical Language Models, SLMs）在自然语言处理（NLP）中用于计算和预测词序列的概率。这些模型通过对大量语料库的统计分析来捕捉语言的结构和模式。

### 1. 词表

假设自然语言处理任务中所有的词的集合为$V$。该集合被称为词表（vocabulary），其中$|V|$表示词表的总长度。

任何一个自然语言文本都可以看作是在该词表中进行采样生成的单词序列。假设有一个词序列$W = (w_1, w_2, ..., w_N)$，其中每个$w_i$都是词表$V$中的一个词，即$w_i \in V$。这个词序列的生成过程可以看作是从词表$V$中依次采样得到的。

例如，在英语词表中，可能包含数十万个单词。对于给定的文本，我们可以认为它是从这个词表中随机选取词语并按照一定顺序排列的结果。

### 2. N-GRAMS模型

N-GRAMS模型是一种常见的统计语言模型，用于确定词序列的概率。假设有一个词序列$W = w_1, w_2, ..., w_N$，模型的任务是计算这个词序列出现的概率$P(W)$。

为了计算词序列的概率，我们可以利用条件概率的概念。假设第N个词的概率依赖于它之前的N-1个词，那么第N个词的条件概率可以表示为$P(w_N|w_{N-1}, ..., w_1)$。这里，前N-1个词被认为是第N个词所依赖的历史信息。

根据概率的乘法公式，我们可以将整个序列的联合概率分解为一系列条件概率的乘积：
$$
P(w_1, w_2, ..., w_N) = P(w_1) \times P(w_2|w_1) \times P(w_3|w_1, w_2) \times ... \times P(w_N|w_{N-1}, ..., w_1)
$$

在实际应用中，考虑到语料库的大小和计算的复杂性，通常会对N的大小进行限制，从而简化模型。常见的简化模型包括一元文法模型、二元文法模型和三元文法模型。

#### （1） 一元文法模型（Unigram Model）

在一元文法模型中，假设文本序列中的每个词都是独立出现的，即当前词的概率不依赖于它之前的词序列。那么词序列的概率可以表示为：
$$
P(w_1, w_2, ..., w_N) = P(w_1) \times P(w_2) \times ... \times P(w_N)
$$
这个模型被称为一元文法模型（Unigram Model）。由于忽略了词与词之间的关系，一元文法模型相对简单，但在捕捉词序列结构方面的效果较差。

#### （2） 二元文法模型（Bigram Model）

在二元文法模型中，假设每个词的概率仅与它之前的一个词有关，即当前词的概率仅依赖于它前面的一个词。那么词序列的概率可以表示为：
$$
P(w_1, w_2, ..., w_N) = P(w_1) \times P(w_2|w_1) \times P(w_3|w_2) \times ... \times P(w_N|w_{N-1})
$$
这个模型被称为二元文法模型（Bigram Model）。二元文法模型考虑了相邻词之间的关系，因此能够捕捉一些简单的词序列模式。

#### （3） 三元文法模型（Trigram Model）

在三元文法模型中，假设每个词的概率仅与它之前的两个词有关，即当前词的概率依赖于它前面的两个词。那么词序列的概率可以表示为：
$$
P(w_1, w_2, ..., w_N) = P(w_1) \times P(w_2|w_1) \times P(w_3|w_1, w_2) \times ... \times P(w_N|w_{N-2}, w_{N-1})
$$
这个模型被称为三元文法模型（Trigram Model）。三元文法模型考虑了更长的历史信息，能够捕捉更复杂的词序列模式。

## 四、问题

### 1、参数空间爆炸与维数灾难

N-gram 模型的一大问题是参数空间爆炸式增长，这也被称为维数灾难（Curse of Dimensionality）。随着 $ n $ 的增加，模型需要处理的词序列组合呈指数级增长，这导致模型参数的数量急剧增加，难以处理更长程的上下文。例如，在 N-gram 模型中，假设我们使用一个 $ n $ 元文法模型（n-gram model）来计算词序列的概率，那么我们需要估计所有可能的 $ n $ 个连续词的联合概率。随着 $ n $ 的增加，可能的 $ n $ 元组合数量会迅速增长。如果词表中有 $ |V| $ 个词，那么我们需要估计的转移概率项数为 $ |V|^n $。这种指数级的增长使得在实际应用中难以获取足够的数据来准确估计所有这些概率，从而导致数据稀疏问题。

### 2、数据稀疏问题及平滑策略

为了缓解数据稀疏问题，需要设计专门的语言模型平滑策略。平滑策略的目的是在面对稀疏数据时，能够对未观察到的事件进行合理的概率估计。常见的平滑方法包括：

- **回退估计（Back-off Estimation）**：在回退估计中，如果 $ n $ 元文法的某个组合在训练数据中未出现，那么我们会“回退”到 $ n-1 $ 元文法来估计该组合的概率。以此类推，直到回退到一元文法。回退估计能够利用较低阶的文法来弥补高阶文法的稀疏性。
  
- **古德-图灵估计（Good-Turing Estimation）**：古德-图灵估计是一种统计方法，用于调整观察到的频率以估计未观察到的事件的概率。具体来说，它通过减少已观察到的事件的概率，并将这部分概率分配给未观察到的事件，从而提供对稀疏数据的平滑估计。

### 3、高阶上下文的刻画能力

尽管这些平滑方法可以缓解数据稀疏问题，它们对于高阶上下文的刻画能力仍然较弱。高阶上下文包含更丰富的语义信息，但由于数据的稀疏性，平滑方法无法精确建模复杂的高阶语义关系。这意味着在处理长距离依赖和复杂语义关系时，高阶 N-gram 模型的效果往往有限。

### 4、词与词之间的内在联系性

N-gram 模型将每个词视为孤立的单元，没有捕捉到词义之间的相似性或联系。例如，如果模型在训练数据中见过“the dog is walking in the bedroom”和“the cat is running in the bedroom”，它应该能够根据“cat”和“dog”以及“walking”和“running”之间的相似性，推测出“the cat is walking in the bedroom”这样的句子概率，但传统的 N-gram 模型做不到这一点。

### 5、One-hot 向量表示的局限

N-gram 模型使用 one-hot 编码来表示词，这种表示方式在数学上将每个词视为一个独立的维度，且每个维度上只有一个词是活跃的（值为 1），其余都是 0。这种表示方式没有包含词的语义信息，且维度非常高，因为词典有多大，向量就有多大。例如，对于一个包含 5 个词的词典：{"I", "love", "nature", "language", "processing"}，每个词对应的 one-hot 向量如下：

- "I" 对应的向量是 [1, 0, 0, 0, 0]
- "love" 对应的向量是 [0, 1, 0, 0, 0]
- "nature" 对应的向量是 [0, 0, 1, 0, 0]
- "language" 对应的向量是 [0, 0, 0, 1, 0]
- "processing" 对应的向量是 [0, 0, 0, 0, 1]

这种表示方法的主要缺点是维度过高，且没有利用词与词之间的相关性。

### 6、现代 NLP 模型的发展

为了克服上述问题，现代 NLP 越来越多地采用神经网络语言模型（如 RNN 和 Transformer）。这些模型能够更好地捕捉复杂的上下文信息并建模高阶语义关系，从而在处理长距离依赖和复杂语义关系时表现得更加出色。尤其是词嵌入方法（如 word2vec），通过将词嵌入到低维空间中，同时保留词与词之间的语义关系，极大地改善了传统 N-gram 模型的局限性。
